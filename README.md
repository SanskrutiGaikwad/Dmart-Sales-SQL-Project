# Dmart-Sales-SQL-Project
DMart Sales & Customer Database System Using MySQL

Overview  
This project designs and implements a relational database system for managing sales transactions, products, and customer information for a retail company like DMart using MySQL.

Business Objective  
Retail businesses require structured databases to track inventory, customer purchases, and revenue efficiently. This project converts raw transactional data into a well-organized SQL database for reporting and analysis.

Database Structure  
- Products table for inventory details  
- Customers table for customer information  
- Sales table for transaction records  
- Primary and foreign key relationships  

Key Features  
- Normalized database design  
- Real-world table relationships  
- SQL queries for sales analysis  
- JOIN operations for reporting  
- Revenue and performance insights  

Tools & Technologies  
MySQL Workbench  

Skills Demonstrated  
Database design, data modeling, SQL querying, and business data analysis.

Project Outcome  
The system enables efficient storage, retrieval, and analysis of retail sales data for business decision-making.

Conclusion  
This project showcases practical SQL expertise in building and analyzing real-world relational databases.
